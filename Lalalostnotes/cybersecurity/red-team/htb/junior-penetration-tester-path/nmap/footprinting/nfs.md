# NFS

`Network File System` (`NFS`) is a network file system developed by Sun Microsystems and has the same purpose as SMB. Its purpose is to access file systems over a network as if they were local

* **NFS Protocol Basics**:
  * Based on **ONC-RPC/SUN-RPC**, using **TCP/UDP port 111**.
  * Utilizes **XDR** for system-independent data exchange.
* **Authentication**:
  * NFS itself lacks authentication or authorization mechanisms.
  * Authentication relies on the **RPC protocol** options.
* **Authorization**:
  * Derived from the file system's information.
  * The **server translates client user info** to match the file system's format.
  * Converts authorization details into **UNIX-compatible syntax**.

**Show Available NFS Shares**

```shell-session
lalalostcode@htb[/htb]$ showmount -e 10.129.14.128

Export list for 10.129.14.128:
/mnt/nfs 10.129.14.0/24    
```

**Mounting NFS Share**

```shell-session
lalalostcode@htb[/htb]$ mkdir target-NFS
lalalostcode@htb[/htb]$ sudo mount -t nfs 10.129.14.128:/ ./target-NFS/ -o nolock
lalalostcode@htb[/htb]$ cd target-NFS
lalalostcode@htb[/htb]$ tree .

.
└── mnt
    └── nfs
        ├── id_rsa
        ├── id_rsa.pub
        └── nfs.share

2 directories, 3 files
```
